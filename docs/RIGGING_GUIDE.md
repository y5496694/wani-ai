# 🐊 와니 캐릭터 Live2D 리깅 가이드

> 악어후드 캐릭터를 Live2D로 움직이게 만드는 **전체 과정**을 초보자도 따라할 수 있도록 정리했습니다.

---

## 📋 전체 로드맵 (예상 소요 시간)

```
Step 1: 도구 준비                    ───  30분
Step 2: 일러스트 파츠 분리 (PSD)      ───  1~2일
Step 3: Live2D 임포트 + 메쉬          ───  2~3시간
Step 4: 디포머 설정                   ───  1일
Step 5: 파라미터 리깅                 ───  2~3일
Step 6: 물리 시뮬레이션               ───  2~3시간
Step 7: 모션/표정 만들기              ───  1일
Step 8: 내보내기 + 테스트             ───  1~2시간
                                    ─────────────
                              합계: 약 1~2주 (초보자 기준)
```

## Step 1: 도구 준비

### 1-1. 필요한 소프트웨어

| 소프트웨어 | 용도 | 가격 |
|-----------|------|------|
| **Clip Studio Paint** 또는 **Photoshop** | 파츠 분리 (PSD 제작) | CSP ₩5,000~/Ps 구독 |
| **Live2D Cubism Editor** | 리깅 & 애니메이션 | **무료** (42일간 PRO 체험) |

### 1-2. Live2D Cubism Editor 설치

1. [live2d.com](https://www.live2d.com/en/cubism/download/editor/) 에서 다운로드
2. 설치 후 실행 → **42일간 PRO 기능** 무료 체험 시작
3. 체험 만료 후에도 FREE 버전으로 계속 사용 가능

### 1-3. FREE vs PRO 제한

| 항목 | FREE | PRO |
|------|------|-----|
| ArtMesh 개수 | 최대 100개 | 무제한 |
| 파라미터 수 | 최대 30개 | 무제한 |
| 텍스처 | 1장, 2048x2048px | 무제한 |

> [!TIP]
> 와니 캐릭터 정도는 **FREE 버전으로 충분**합니다.
> 다만 42일 PRO 체험기간 동안 작업하면 더 편리한 기능을 쓸 수 있으니, 파츠분리를 먼저 끝내고 Cubism을 설치하세요.

## Step 2: 일러스트 파츠 분리

이 단계가 **가장 중요하고 시간이 많이 걸리는** 단계입니다.

> 📌 **파츠 분리 참고 이미지**: `docs/parts_separation_guide.png` 를 참고하세요.

### 2-1. 핵심 원칙

1. **한 파츠 = 한 레이어**: 라인 + 색칠 + 그림자를 하나로 병합
2. **숨겨진 부분 그리기**: 머리카락 뒤의 두피, 후드 아래의 머리카락 등 "보이지 않지만 움직이면 보일 부분"을 미리 그려야 함
3. **충분히 크게**: 잘린 경계가 움직임에서 보이지 않도록 파츠를 넉넉하게 그리기

#### 작업 순서 (추천)

1. **얼굴부터** → 눈, 입, 코를 분리 (가장 중요, 가장 많이 움직임)
2. **머리카락** → 앞머리, 옆머리, 뒷머리로 분리
3. **악어후드** → 후드 본체, 악어 눈, 이빨 분리
4. **몸통** → 후디, 팔, 손
5. **꼬리** → 3등분 (근부, 중간, 끝)
6. **하체** → 스커트, 허벅지, 니삭스, 구두

### 2-4. PSD 내보내기 체크리스트

- [ ] 파일 형식: `.psd`
- [ ] 색상 모드: **RGB**
- [ ] 비트 깊이: **8bit/channel**
- [ ] 색상 프로파일: **sRGB**
- [ ] 레이어 마스크: 모두 **적용(병합)** 완료
- [ ] 블렌딩 모드: **Normal** 만 사용 (Multiply, Overlay 등 피하기)
- [ ] 같은 이름의 레이어 없음
- [ ] 빈 레이어 없음

## Step 3: Live2D 임포트 + 메쉬 설정

### 3-1. PSD 임포트

1. Live2D Cubism Editor 실행
2. **파일 → 신규 → 모델** 선택
3. PSD 파일을 에디터 화면에 **드래그 앤 드롭**
4. 레이어들이 자동으로 ArtMesh로 변환됨

### 3-2. 드로우 순서 정리

임포트 후 레이어 겹침 순서를 확인하고 정리:

```
앞 ←──────────────────────────→ 뒤

hair_bangs  >  eyes  >  face_base  >  hood  >  hair_back
mouth       >  nose  >  neck       >  body
shoe        >  sock  >  thigh      >  skirt
```

### 3-3. 메쉬 편집

각 ArtMesh에 메쉬(격자점)를 설정합니다. **잘 움직여야 할 파츠일수록 촘촘한 메쉬가 필요합니다.**

| 파츠 | 메쉬 밀도 | 설명 |
|------|----------|------|
| 눈 (iris, pupil, eyelid) | ★★★ 고밀도 | 가장 많이 변형됨 |
| 입 (upper, lower) | ★★★ 고밀도 | 립싱크에 필수 |
| 눈썹 | ★★☆ 중밀도 | 감정 표현에 중요 |
| 앞머리 가닥 | ★★☆ 중밀도 | 흔들림 애니메이션 |
| 꼬리 | ★★☆ 중밀도 | 감정 표현 (흔들기) |
| 악어후드 | ★★☆ 중밀도 | 머리와 함께 움직임 |
| 후디 본체 | ★☆☆ 저밀도 | 큰 변형 없음 |
| 스커트/니삭스 | ★☆☆ 저밀도 | 약간의 흔들림만 |

**메쉬 편집 방법:**
1. ArtMesh 선택
2. 상단 메뉴 → **메쉬 자동생성** 클릭
3. 밀도 슬라이더로 조절 (눈/입은 높게, 몸통은 낮게)
4. **수동 편집**: 변형이 깨지는 부분이 있으면 점을 직접 추가/이동

## Step 4: 디포머 설정

디포머(Deformer)는 **여러 ArtMesh를 한꺼번에 움직이는 "뼈대"** 입니다.

### 4-1. 디포머 계층 구조

```
📦 Root
├── 🔄 D_Body (회전/단순 디포머 — 몸 전체 기울기)
│   │
│   ├── 📐 D_Head (워프 디포머 — 머리 회전 XYZ)
│   │   ├── 📐 D_Face (워프 — 얼굴 변형)
│   │   │   ├── 📐 D_Eye_L (워프 — 왼쪽 눈 전체)
│   │   │   │   ├── ArtMesh: eyelash_upper_L
│   │   │   │   ├── ArtMesh: eyelid_L
│   │   │   │   ├── ArtMesh: eye_white_L
│   │   │   │   ├── ArtMesh: eye_iris_L
│   │   │   │   ├── ArtMesh: eye_pupil_L
│   │   │   │   └── ArtMesh: eye_highlight_L
│   │   │   ├── 📐 D_Eye_R (워프)
│   │   │   ├── 📐 D_Brow_L (워프 — 눈썹)
│   │   │   ├── 📐 D_Brow_R (워프)
│   │   │   ├── 📐 D_Mouth (워프 — 입 전체)
│   │   │   ├── ArtMesh: nose
│   │   │   ├── ArtMesh: cheek_blush
│   │   │   └── ArtMesh: face_base
│   │   │
│   │   ├── 📐 D_Hood (워프 — 악어후드)
│   │   │   ├── ArtMesh: hood_upper
│   │   │   ├── ArtMesh: hood_teeth
│   │   │   ├── ArtMesh: hood_eye_L
│   │   │   ├── ArtMesh: hood_eye_R
│   │   │   └── ArtMesh: hood_base
│   │   │
│   │   ├── 📐 D_Hair_Front (워프 — 앞머리 흔들림)
│   │   ├── 📐 D_Hair_Side_L (워프)
│   │   ├── 📐 D_Hair_Side_R (워프)
│   │   └── 📐 D_Hair_Back (워프)
│   │
│   ├── 📐 D_Hoodie (워프 — 후디 본체)
│   ├── 🔄 D_Arm_L (회전 — 팔)
│   ├── 🔄 D_Arm_R (회전)
│   ├── 📐 D_Tail (워프 — 꼬리) ⭐
│   └── 📐 D_Skirt (워프 — 스커트)
```

### 4-2. 디포머 종류

| 아이콘 | 종류 | 용도 | 사용 위치 |
|--------|------|------|----------|
| 📐 | **워프 디포머** | 격자로 변형 (찌그러뜨리기, 늘리기) | 얼굴, 눈, 입, 머리카락, 꼬리 |
| 🔄 | **회전 디포머** | 축을 중심으로 회전 | 고개, 팔, 몸 기울기 |

### 4-3. 디포머 만드는 법

1. 자식이 될 ArtMesh(들) 선택
2. 상단 메뉴 → **워프 디포머 생성** 또는 **회전 디포머 생성**
3. 이름 입력 (예: `D_Eye_L`)
4. 부모 디포머 선택 (예: `D_Face`)
5. 격자 크기 설정 (워프: 보통 3×3 또는 4×4)

> [!WARNING]
> 디포머의 **부모-자식 관계**가 가장 중요합니다!
> 머리(D_Head)를 돌리면 그 안의 눈, 입, 머리카락이 모두 같이 돌아가야 합니다.

## Step 5: 파라미터 리깅 (핵심!)

파라미터에 **키폼(KeyForm)**을 설정해서 캐릭터를 움직이게 만듭니다.

### 5-1. 필수 파라미터 목록

와니 캐릭터에 필요한 파라미터를 **우선순위 순**으로 정리:

#### 🥇 1순위 — 반드시 필요 (코드의 립싱크/눈깜빡임 연동)

| 파라미터 ID | 이름 | 범위 | 용도 |
|------------|------|------|------|
| `ParamEyeLOpen` | 왼눈 열기 | 0~1 | 눈 깜빡임 |
| `ParamEyeROpen` | 오른눈 열기 | 0~1 | 눈 깜빡임 |
| `ParamMouthOpenY` | 입 열기 | 0~1 | **립싱크 (코드 연동)** |
| `ParamMouthForm` | 입 형태 | -1~1 | 웃음(+) / 슬픔(-) |

#### 🥈 2순위 — 생동감에 필수

| 파라미터 ID | 이름 | 범위 | 용도 |
|------------|------|------|------|
| `ParamAngleX` | 머리 좌우 | -30~30 | 고개 좌우 돌리기 |
| `ParamAngleY` | 머리 상하 | -30~30 | 고개 끄덕이기 |
| `ParamAngleZ` | 머리 기울기 | -30~30 | 고개 갸우뚱 |
| `ParamBodyAngleX` | 몸 좌우 | -10~10 | 몸 기울기 |
| `ParamBreath` | 호흡 | 0~1 | 몸 미세 상하 움직임 |

#### 🥉 3순위 — 감정 표현

| 파라미터 ID | 이름 | 범위 | 용도 |
|------------|------|------|------|
| `ParamBrowLY` | 왼눈썹 높이 | -1~1 | 놀람(+) / 분노(-) |
| `ParamBrowRY` | 오른눈썹 높이 | -1~1 | 놀람(+) / 분노(-) |
| `ParamEyeBallX` | 시선 좌우 | -1~1 | 시선 이동 |
| `ParamEyeBallY` | 시선 상하 | -1~1 | 시선 이동 |
| `ParamCheek` | 볼 홍조 | 0~1 | 부끄러움 표현 |
| `ParamEyeLSmile` | 왼눈 웃음 | 0~1 | 눈웃음 |
| `ParamEyeRSmile` | 오른눈 웃음 | 0~1 | 눈웃음 |

#### 🐊 보너스 — 와니 전용!

| 파라미터 ID | 이름 | 범위 | 용도 |
|------------|------|------|------|
| `ParamTailSwing` | 꼬리 흔들기 | -1~1 | **기쁠 때 빠르게, 슬플 때 축 처짐** |
| `ParamHoodBounce` | 후드 흔들림 | -1~1 | 머리 움직임에 따라 |
| `ParamHairSwing` | 머리카락 흔들림 | -1~1 | 물리 시뮬로 자동 |

### 5-2. 파라미터에 키폼 설정하는 법

가장 중요한 **눈 깜빡임**을 예시로 설명:

```
Step 1: ParamEyeLOpen 파라미터 선택
Step 2: 슬라이더를 "1" (눈 뜨기)에 놓기
Step 3: 키폼 추가 (● 버튼)
Step 4: 눈이 떠진 상태로 ArtMesh 배치 (기본 상태 = 변경 없음)

Step 5: 슬라이더를 "0" (눈 감기)으로 이동
Step 6: 키폼 추가
Step 7: eyelid_L ArtMesh를 아래로 내려서 눈을 덮기
        eye_white, iris, pupil, highlight를 축소/숨기기

완료! 이제 슬라이더를 움직이면 눈이 깜빡임!
```

### 5-3. 리깅 순서 (추천)

```
1️⃣  눈 깜빡임 (ParamEyeLOpen / ParamEyeROpen)
    └→ 가장 기본, 성취감 높음

2️⃣  입 열기/닫기 (ParamMouthOpenY)
    └→ 립싱크의 핵심

3️⃣  입 웃음/슬픔 (ParamMouthForm)
    └→ 감정 표현 기초

4️⃣  머리 좌우 회전 (ParamAngleX)
    └→ 생동감의 핵심. 워프 디포머로 얼굴 전체 변형
        Tip: "코너 합성" 기능 활용 (AngleX × AngleY 조합)

5️⃣  머리 상하/기울기 (ParamAngleY / Z)

6️⃣  눈썹 (BrowLY / BrowRY)

7️⃣  시선 (EyeBallX / Y)

8️⃣  호흡 (ParamBreath)
    └→ 몸을 약간 위아래로

9️⃣  꼬리 흔들기 (ParamTailSwing) ⭐
    └→ 와니 전용! 꼬리 워프 디포머를 좌우로 흔들기

🔟  머리카락/후드 흔들림 → Step 6 물리에서 자동 처리
```

### 5-4. 머리 회전 리깅 팁 (가장 어려운 부분)

```
ParamAngleX (좌우 회전) 리깅:

값: -30 (왼쪽)         0 (정면)         +30 (오른쪽)
    ┌───────┐          ┌───────┐         ┌───────┐
    │ 👁️  👁️│          │ 👁️ 👁️ │         │👁️  👁️ │
    │   👃  │          │  👃   │         │  👃   │
    │   👄  │          │  👄   │         │   👄  │
    └───────┘          └───────┘         └───────┘
    
    얼굴 전체가            기본              얼굴 전체가
    오른쪽으로 쏠림                          왼쪽으로 쏠림
    왼쪽 눈 작아짐                          오른쪽 눈 작아짐
    코/입 왼쪽 이동                          코/입 오른쪽 이동
```

> [!TIP]
> **"코너 합성" 기능**을 활용하세요!
> AngleX와 AngleY를 각각 리깅하면, 에디터가 대각선 방향(왼위, 오른아래 등)을 자동 보간해줍니다.
> 상단 메뉴 → **모델링 → 코너의 폼 생성**

## Step 6: 물리 시뮬레이션

물리 설정으로 머리카락, 꼬리, 후드가 **자연스럽게 흔들리게** 만듭니다.

### 6-1. 물리 설정 열기

모델링 → **물리연산/씬 블렌드 설정** 에서 열기

### 6-2. 물리 그룹 설정

| 그룹 이름 | 입력 | 출력 | 설정 |
|----------|------|------|------|
| 앞머리 흔들림 | ParamAngleX, Z | ParamHairSwing (앞머리 각 가닥) | 길이: 중, 반동: 약 |
| 옆머리 흔들림 | ParamAngleX, Z | hair_side L/R 변형 | 길이: 장, 반동: 중 |
| **꼬리 흔들림** ⭐ | ParamAngleX, ParamTailSwing | tail 각 파트 변형 | 길이: 장, 반동: 강 |
| 후드 흔들림 | ParamAngleY | hood 변형 | 길이: 단, 반동: 약 |

### 6-3. 물리 설정 팁

- **진자 길이**: 길수록 천천히, 짧수록 빠르게 흔들림
- **반동**: 높을수록 계속 흔들임 (꼬리에 적합), 낮으면 빨리 멈춤
- **공기 저항**: 높으면 움직임이 부드러워짐
- 꼬리는 **3단 물리 체인** (근부 → 중간 → 끝) 으로 만들면 자연스러움

## Step 7: 모션/표정 만들기

### 7-1. 표정 (Expression) 파일

감정별 표정을 `.exp3.json` 파일로 만듭니다:

| 파일명 | 감정 | 설정 내용 |
|--------|------|----------|
| `neutral.exp3.json` | 평온 | 기본 상태 |
| `happy.exp3.json` | 기쁨 | EyeLSmile=1, EyeRSmile=1, MouthForm=1 |
| `sad.exp3.json` | 슬픔 | BrowLY=-0.5, BrowRY=-0.5, EyeOpen=0.5, MouthForm=-0.5 |
| `surprise.exp3.json` | 놀람 | EyeOpen=1.3, MouthOpenY=0.5, BrowY=1 |
| `angry.exp3.json` | 분노 | BrowAngle=-1, MouthForm=-0.5 |
| `shy.exp3.json` | 부끄러움 | Cheek=1, EyeBallX=-0.3, EyeOpen=0.6 |

**만드는 법**: Cubism Editor → **표정 뷰** → 파라미터 조절 → 저장

### 7-2. 모션 (Motion) 파일

| 파일명 | 용도 |
|--------|------|
| `idle.motion3.json` | 기본 대기 (호흡 + 미세 움직임) |
| `happy.motion3.json` | 기쁨 (꼬리 빠르게 흔들기) |
| `nod.motion3.json` | 끄덕이기 (리액션) |
| `tilt.motion3.json` | 고개 갸우뚱 (생각 중) |

**만드는 법**: Cubism Editor → **애니메이션 뷰** → 타임라인에서 키프레임 설정 → 내보내기

## Step 8: 내보내기 + 와니 AI 연동

### 8-1. 내보내기

1. Cubism Editor → **파일 → 내장용 파일 내보내기 → moc3 파일 내보내기**
2. 설정:
   - SDK 타겟: **SDK for Native**
   - 텍스처 아틀라스: 2048×2048 (FREE 제한)
3. 내보내기 결과물:

```
models/wani/
├── wani.model3.json     ← 모델 정의 (이 파일이 진입점)
├── wani.moc3            ← 컴파일된 모델 데이터
├── wani.physics3.json   ← 물리 시뮬레이션 설정
├── textures/
│   └── texture_00.png   ← 텍스처 아틀라스
├── motions/
│   ├── idle.motion3.json
│   ├── happy.motion3.json
│   └── ...
└── expressions/
    ├── neutral.exp3.json
    ├── happy.exp3.json
    └── ...
```

### 8-2. 와니 AI에 연동

내보낸 파일을 프로젝트의 `models/wani/` 디렉토리에 복사하면 자동으로 인식됩니다!

```bash
# Pi에서 테스트
cd ~/wani-ai
./scripts/start.sh

# Live2D 모델이 감지되면:
# "Live2D 모델 발견 — Live2D 렌더러 사용" 로그 출력
```

## ❓ FAQ

**Q: 파츠 분리가 너무 어려워요**
> A: 원본 일러스트를 직접 자르지 말고, 새 레이어에 **처음부터 파츠별로 다시 그리는 것**이 더 깔끔합니다. 원본은 아래에 깔아두고 참고하세요.

**Q: FREE 버전 제한(30 파라미터)에 걸려요**
> A: 위 목록에서 1순위 + 2순위 파라미터만 해도 약 14개, 3순위까지 합쳐도 ~25개입니다. 와니 전용 파라미터 빼면 충분합니다.

**Q: 외주 맡기고 싶어요**
> A: 크몽에서 "Live2D 리깅"으로 검색하시면 됩니다. PSD 파일 + 이 가이드의 파라미터 목록을 전달하세요. 가격: 약 10~30만원.

**Q: Live2D 말고 더 쉬운 방법?**
> A: 지금 코드에 **스프라이트 폴백**이 이미 구현되어 있습니다! 캐릭터 이미지를 `models/wani/wani_sprite.png`로 넣으면 호흡 애니메이션 + 상태 표시가 동작합니다. Live2D는 나중에 추가해도 됩니다.
